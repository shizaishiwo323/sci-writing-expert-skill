# Terminology and Definition Control

## 1. Terminology hierarchy

Prefer:

1. an explicit user glossary;
2. terms already defined in the manuscript;
3. terminology established in target articles;
4. a recognized discipline-standard term;
5. a plain descriptive phrase marked `author-confirm`.

Do not invent a compact technical label merely to sound academic.

## 2. Definition timing

Define a term when:

- it is central to the research question;
- it has a manuscript-specific meaning;
- multiple field meanings exist;
- an abbreviation will recur;
- readers cannot interpret a result without it.

Do not define common field terms unnecessarily for a specialist audience.

## 3. Abbreviations

- define at first use subject to journal policy;
- avoid abbreviations used only once or twice;
- preserve capitalization and plural form;
- never redefine an abbreviation differently;
- check Abstract and main text rules separately.

## 4. Consistency

Use one term for one concept unless a distinction is intended.

High-risk pairs:

- model / framework / scheme;
- accuracy / precision;
- sample / specimen;
- effect / response;
- validation / verification;
- correlation / relationship / causation;
- parameter / variable;
- sensitivity / responsiveness.

These are not merely stylistic synonyms.

## 5. New compounds

Before retaining a coined term, ask:

- Is it established?
- Is it defined?
- Does it distinguish a real concept?
- Will the same form recur?
- Could readers misparse the relation among parts?

Use `dash_policy.md` for detailed compound guidance.

## 6. Translation uncertainty

```text
Preferred term: ...
Alternative: ...
Difference: ...
author-confirm: ...
```

Do not hide ambiguity with a vague word.
