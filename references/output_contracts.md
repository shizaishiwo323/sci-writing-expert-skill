# Output Contracts

Match output depth to the task.

## Sentence polish

```text
Revised version:
...

Main changes:
- ...
- ...

Meaning preserved:
Yes / Potential issue: ...
```

Omit notes when the user asks for text only.

## Chinese-to-English translation

```text
Translation:
...

Notes:
- terminology or scientific ambiguity, only if material;
- claim-strength choice, only if material.
```

## Paragraph revision

```text
Paragraph function:
...

Revised paragraph:
...

Logic changes:
- ...

Author confirmation:
- ...  [only when needed]
```

## Section diagnosis and revision

```text
Section move map:
...

Main structural issues:
1. ...

Recommended order:
...

Revised section:
...

Unresolved scientific questions:
- ...
```

For long sections, work in requested scope or batches.

## Manuscript audit

```text
Central story:
...

Critical issues:
1. ...

Section-level findings:
...

Promise-payoff and evidence-chain gaps:
...

Priority revision plan:
1. ...
2. ...
3. ...

Safe to line-edit now:
...
```

```text
Preferred:
...

...

...
```

## Change labels

Use sparingly:

- `[meaning]`
- `[logic]`
- `[structure]`
- `[cohesion]`
- `[grammar]`
- `[claim strength]`
- `[terminology]`
- `[journal fit]`
- `[author-confirm]`

## Explanation length

- phrase/sentence: 0-4 concise notes;
- paragraph: 3-7 material notes;
- section: structured diagnosis;
- manuscript: prioritized audit.

Do not explain every article or comma unless requested.
