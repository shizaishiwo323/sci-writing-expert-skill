# Manuscript Architecture Audit

## Purpose

Diagnose a draft before sentence-level polishing.

## 1. One-sentence research story

Complete:

```text
Because [problem/gap], this study [action] to determine [question],
finding [central result], which [bounded contribution].
```

This is a diagnostic, not necessarily manuscript text. If it cannot be completed from the draft, identify what is missing.

## 2. Promise-payoff matrix

| Promise | Introduction/Abstract | Method that tests it | Result that answers it | Discussion payoff |
|---|---|---|---|---|

Check for:

- an Introduction promise with no Result;
- a Result not prepared by a question or method;
- a Discussion claim unsupported by Results;
- an Abstract claim absent from the main text;
- a Title broader than the evidence.

## 3. Section-function audit

### Title

Does it accurately define subject, relation, and scope?

### Abstract

Does it contain aim/action, method, key result, and bounded meaning?

### Introduction

Does it move from context through literature and gap to the present study?

### Methods

Does every major result have a reproducible evidence pathway?

### Results

Are findings ordered to answer the question rather than follow raw analysis chronology?

### Discussion

Does interpretation map back to the gap and distinguish evidence from speculation?

### Conclusion

Does it close the argument without new claims?

## 4. Paragraph map

| Section | Paragraph | Function | Main claim | Evidence/source | Link to next |
|---|---:|---|---|---|---|

Flag:

- duplicate functions;
- mixed-function paragraphs;
- paragraphs with no main claim;
- claims with no evidence;
- evidence with no interpretation;
- abrupt topic shifts;
- unsupported transitions.

## 5. Evidence-chain audit

For each central claim:

```text
question -> method -> observation -> analysis -> interpretation -> limitation
```

Identify missing or weak links.

## 6. Figure/table storyline

For each main visual:

- question answered;
- one-sentence result;
- why it appears here;
- later claim that depends on it.

Visual order should support the argument, not merely analysis chronology.

## 7. Results-Discussion boundary

Check whether:

- Results contains excessive literature review or speculation;
- Discussion repeats Results without interpretation;
- mechanisms appear before relevant observations;
- limitations are detached from the claims they limit.

Respect combined-section conventions.

## 8. Severity

- **Critical:** meaning, validity, reproducibility, or central logic.
- **Major:** blocks understanding or reviewer confidence.
- **Moderate:** weakens flow or function.
- **Minor:** local style or consistency.

Prioritize critical and major issues before polishing.

