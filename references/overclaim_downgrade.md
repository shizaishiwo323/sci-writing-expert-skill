# Overclaim Control

## Core rule

Preserve the source's evidence strength. Downgrade unsupported promotion; do not automatically weaken a strong, justified claim.

## Never strengthen automatically

- `suggests` -> `demonstrates`;
- `indicates` -> `proves`;
- `is consistent with` -> `confirms`;
- `may`, `might`, or `could` -> certainty;
- a local result -> universal or field-scale conclusion.

## Verb calibration

| Use cautiously | Safer when evidence is limited |
|---|---|
| prove | show, indicate, support |
| demonstrate | show, indicate |
| establish | support, define, provide |
| reveal | show, indicate |
| confirm | support, is consistent with |
| validate | compare with, benchmark against |
| enable | allow, may support |
| predict | estimate, project, suggest |
| govern | influence, contribute to, associate with |

Choose by actual evidence, not by a mechanical replacement table.

## Intensifiers

Usually delete unless quantified or explicitly supported:

- `remarkably`;
- `critically`;
- `fundamentally`;
- `dramatically`;
- `robustly`;
- `clearly`;
- `uniquely`.

## Adjectives

Question:

- `novel`;
- `unprecedented`;
- `transformative`;
- `powerful`;
- `universal`;
- `comprehensive`;
- `robust`.

Replace with a concrete property where possible.

## Scope

Use boundaries such as:

- `under these conditions`;
- `in the simulations`;
- `for the tested samples`;
- `within the measured range`.

Do not add a scope marker that changes the intended claim; use `author-confirm` if uncertain.

## Evidence labels

- direct observation: `shows`;
- supported interpretation: `indicates`;
- plausible indirect interpretation: `suggests`;
- consistency with external evidence: `is consistent with`;
- extrapolated implication: `may`;
- formal proof: `proves`, only when genuinely formal.
